1. Discuss the scope of var, let, and const

VAR

Scope: Variables declared with var are scoped to the function in which they are defined. They cannot be accessed outside of this function.

Hoisting: When variables are declared with var, they are hoisted to the top of their containing function. This means they can be accessed before they are actually declared, but their value will be undefined until they are initialized.

LET

Scope: Variables declared with let are scoped to the block in which they are defined. This could be within a function, loop, or any code block enclosed by curly braces.

Hoisting: let variables are also hoisted, but unlike var, they are not initialized. This means you cannot access them before they are declared within the block.

CONST

Scope: Similar to let, variables declared with const are block-scoped.

Immutability: const variables are intended to be constants, meaning their value cannot be reassigned after initialization. However, for objects and arrays declared with const, the properties or elements within them can still be modified.

2. Tell us the use cases of null and undefined

NULL

Represents Absence: null signifies the deliberate absence of any object value or value in general. It's used to indicate that a variable or property should have no value or has been intentionally cleared.

Placeholder: Can serve as a placeholder or initial value that might be assigned later.

Return Value: Some functions or methods may return null to indicate a lack of meaningful result.

UNDEFINED

Default Initialization: Variables declared but not assigned a value are automatically initialized with undefined. Often used as the default value for function parameters and variables.

Property Access: Accessing an object property that doesn't exist results in undefined.

Array Elements: Accessing an index beyond the array's length yields undefined.

Function Returns: Functions not explicitly returning a value return undefined.

3. What do you mean by REST API?

A REST API, or Representational State Transfer Application Programming Interface, is a software architectural style for designing networked applications. It operates over the Hypertext Transfer Protocol (HTTP) and is based on the concept of resources, which are typically identified by Uniform Resource Identifiers (URIs).

REST APIs enable communication between client and server applications by allowing clients to perform various operations on resources, such as retrieving, updating, creating, or deleting them, using standard HTTP methods like GET, POST, PUT, DELETE, etc. This architectural style emphasizes scalability, simplicity, and the uniform interface between components, making it widely adopted for building web services and web-based applications.
